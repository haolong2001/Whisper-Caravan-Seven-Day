# v0.3 Scope

## Intent

Make different NPCs react differently based on what they are allowed to know, what they trust, and how they relate to the player.

## Expected Files to Touch

Likely:

- `lib/types.ts`
- `lib/mockData.ts`
- `lib/gameLogic.ts`
- `app/page.tsx`
- `components/EvidencePanel.tsx`
- `components/MemoryPanel.tsx`

Avoid touching unrelated files unless necessary.

## Architecture Constraints

- Keep NPC profile data in `lib/mockData.ts`.
- Keep NPC access and reaction rules in `lib/gameLogic.ts`.
- Keep shared types in `lib/types.ts`.
- Keep React components presentational where possible.
- Do not put NPC branching logic directly inside JSX.

## Scope

- multiple NPC profiles
- per-NPC memory access rules
- per-NPC reliability thresholds
- faction-aware reaction differences
- deterministic client-side reactions for multiple NPCs

## Required Behavior

- Add multiple NPC profiles.
- Each NPC should define:
  - `id`
  - `name`
  - `faction`
  - `acceptedMemoryTypes`
  - `rejectedMemoryTypes`
  - `minReliability`
  - `visibleMemoryScopes`
- NPCs should not all see the same memories.
- NPC reactions should be based on:
  - active memories
  - visibility
  - memory type
  - reliability
  - faction relationship
- Add deterministic client-side reactions for:
  - Deer Guard
  - Fox Merchant
  - Crow Broker
  - Bear Judge

## Acceptance Criteria

- [ ] Deer Guard, Fox Merchant, Crow Broker, and Bear Judge each have explicit NPC profiles.
- [ ] NPC profiles include accepted memory types, rejected memory types, visible memory scopes, and minimum reliability.
- [ ] NPCs do not all receive the same evidence.
- [ ] NPC reactions only use active memories.
- [ ] NPC reactions respect memory visibility.
- [ ] NPC reactions respect memory type.
- [ ] NPC reactions respect reliability thresholds.
- [ ] NPC reactions are affected by faction relationship where appropriate.
- [ ] The UI explains why each NPC accepted or rejected evidence.
- [ ] v0.2 Day 1 → Day 8 loop still works.
- [ ] `npm run build` passes.

## Non-Goals

- No LLM
- No vector database
- No full backend
- No free-form NPC chat
- No API routes
- No auth
- No save/load
- No combat
- No inventory
- No map screen
