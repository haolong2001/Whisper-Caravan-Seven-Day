# v0.3 Scope

## Intent

Make different NPCs react differently based on what they are allowed to know, what they trust, and how they relate to the player.

## Scope

- multiple NPC profiles
- per-NPC memory access rules
- per-NPC reliability thresholds
- faction-aware reaction differences
- deterministic client-side reactions for multiple NPCs

## Required Behavior

- Add multiple NPC profiles.
- Each NPC should define:
  - `id`
  - `name`
  - `faction`
  - `acceptedMemoryTypes`
  - `rejectedMemoryTypes`
  - `minReliability`
  - `visibleMemoryScopes`
- NPCs should not all see the same memories.
- NPC reactions should be based on:
  - active memories
  - visibility
  - memory type
  - reliability
  - faction relationship
- Add deterministic client-side reactions for:
  - Deer Guard
  - Fox Merchant
  - Crow Broker
  - Bear Judge

## Success Criteria

- Different NPCs can produce different outcomes from the same run.
- The player can understand why one NPC reacted differently from another.
- Access rules are explicit enough to migrate later into backend retrieval filters.

## Non-Goals

- No LLM
- No vector database
- No full backend
- No free-form NPC chat
